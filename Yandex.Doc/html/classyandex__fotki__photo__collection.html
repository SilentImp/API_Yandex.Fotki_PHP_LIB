Добавляет ноую фотографию в коллекцию.

Аргументы:
path путь к файлу, содержащему изображения
pub_channel Уникальное имя-маркер клиентского приложения, осуществляющего загрузку фотографии.
app_platform Дополнительная информация о платформе клиентского приложения, осуществляющего загрузку.
app_version Версия клиентского приложения, осуществляющего загрузку.
title Название фотографии. Не может быть пустой строкой.
tags Массив содержащий теги (метки), уточняющие смысл фотографии.
yaru Флаг публикации фотографии на странице пользователя на Я.ру. Допустимые значения: "1" - опубликовать (по умолчанию); "0" - не опубликовывать.
access_type Уровень доступа к фотографии. Допустимые значения: public (по умолчанию) - для всех; friends - для друзей; private - для себя.
album Идентификатор альбома для загрузки фотографии. Альбом должен существовать.
disable_comments Флаг запрета комментариев. Значение по умолчанию: false.
xxx Флаг «только для взрослых» (можно только установить, снять нельзя). Значение по умолчанию: false.
hide_orig Флаг запрета публичного доступа к оригиналу фотографии. Значение по умолчанию: false. Если данный флаг установлен в true, автор не сможет получить оригинал фотографии при помощи API Фоток. Для этого нужно воспользоваться возможностями сервиса Яндекс.Фотки.
storage_private Флаг, закрывающий доступ к фотографии по URL со страниц вне домена Яндекс.Фоток. Значение по умолчанию: false.
token токен, подтверждающий аутентификацию пользователя. Обязательный аргумент.
Возвращает:
ассоциативный массив. Если yaru==0, то возвращается array('image_id'=>photo_id), где {photo_id} - численный идентификатор фотографии. Если yaru==1 то возвращается array('image_id'=>photo_id,'post_id'=>post_id), где {photo_id} - идентификатор фотографии, а {post_id} - идентификатор поста на Я.ру.

Осуществляет поиск по коллекции фотографий с заданным заголовком.

Аргументы:
title название фотографии. Обязательный аргумент.
limit максимально допустимое количество элементов выборки. Если установлено, то по достижении указанного числа найденных фотографий поиск будет завершен. В противном случае будут проверены все альбомы выборки на всех страницах. Если равно 0, то игнорируется.
Возвращает:
FALSE если альбомов с таким названием не найдено, альбом, если найдено единственное соответствие и массив альбомов, если найдено более одного вхождения.

search ( $  order = "updated",
$  offset_time = null,
$  offset_id = "",
$  limit = 100,
$  token = null 
)

Выполняет поиск по коллекции с заданными условиями.

Аргументы:
order Порядок отображения элементов выдачи. Допустимые значения: updated (по умолчанию) - по времени последнего изменения, от новых к старым; rupdated - по времени последнего изменения, от старых к новым; published - по времени загрузки (для фотографии) или создания (для альбома), от новых к старым; rpublished - по времени загрузки (для фотографии) или создания (для альбома), от старых к новым; created - по времени создания согласно EXIF-данным, от новых к старым; rcreated - по времени создания согласно EXIF-данным, от старых к новым.
offset_time Время создания ресурса в формате UTC с точностью до секунды. Исключение: ссылки с order равным created или rcreated, в которых время указывается без часового пояса.
offset_id Численный идентификатор ресурса на Яндекс.Фотках.
limit Количество элементов на странице выдачи.
token Токен, подтверждающий аутентификацию пользователя. Если не задан, используется токен, который был передан конструктору. Если не задан и он, то метод вызовет исключение.
